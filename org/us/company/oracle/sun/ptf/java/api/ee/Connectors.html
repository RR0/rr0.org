J. C. A.

Java Connector Architecture : Architecture de connecteurs Java.

Egalement appelée "J2C", "J2EE Connectors", "J2CA" ou "JCX". L'utilisation du terme JCA introduit malheureusement une confusion avec la Java Cryptography Architecture.

Besoin

  • Permettre aux fournisseurs d'EIS de s'intégrer avec des environnements Java de manière standard ("plug") ;
  • Permettre aux applications Java d'exploiter des EIS manière standard ("play") ;

Analyse

JCA est la solution Java d'intégration des EIS. Elle peut être utilisée en environnement :

  • non géré : il s'agit généralement d'une application légère
  • géré : il s'agit d'une application déployée dans un serveur applicatif. Les connecteurs définissent alors les contrats de gestion :
    • des connexions :
    • des transactions :
    • de la sécurité :
      L'accès à l'EIS peut se faire directement où via un cache.

Conception

JCA est intégrée dans J2EE 1.3.

Implémentation

Tout connecteur doit implémenter les interfaces standards du package javax.resource.

Une API est dédiée aux outils manipulant des connecteurs de manière générique : CCI (Common Client Interface).

Déploiement

  • Un connecteur est livré dans une archive Java (.RAR) contenant :
    • un adaptateur de ressource (Resource Adapter), i.e. une implémentation des interfaces standard des connecteurs.
    • un descripteur (ra.xml)

Limitations

  • La communication avec l'EIS est synchrone : corrigé dans une future version ;
  • La communication avec l'EIS est undidirectionnelle (application->EIS) : corrigé dans une future version.
