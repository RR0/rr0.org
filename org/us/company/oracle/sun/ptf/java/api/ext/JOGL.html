Conception

A la livraison, JOGL ne comporte que 150 lignes de code natif (dont 100 sont destinées à contourner des bugs des pilotes OpenGL les plus anciens sur Windows). Le reste du code natif est auto-généré au cours du processus de construction par un GlueGen.

L'affichage s'effectue sur un GLCanvas, dont les événements du cycle de vie peuvent être reçus sur un GLEventListener, tels que :

  • init() : appelée une seule fois
  • display() : à chaque fois qu'un affichage du canvas est requis
  • reshape() : à chaque qu'un redimensionnement du canvas est requis
  • displayChanged() : quand la configuration video a changé

Un Animator permet d' encapsuler des appels en boucle à display() tout en évitant de surcharger le CPU.
