composants web
.

Besoin

Définir un standard de composants web.

Analyse

Il existe 2 types de web components :

  • Autonomes, définissant leur propre nouveau tag
  • Personnalisés, étendant un tag existant (built in).

Parce que supportés par l'ensemble des navigateurs web, les composants web ne nécessitent pas de librarie ni framework spécifiques.

En tant que composants, les web components permettent une encapsulation de leur contenu et de son style

Conception

Les web components nécessitent :

  • des éléments personnalisés (custom elements) pour définir ses propres balises ;
  • Un DOM "fantôme" (shadow DOM) pour stocker son contenu et son style (éventuellement via des modèles HTML ou HTML templates).

Comme toute balise HTML, un élément personnalisé peut avoir des éléments fils, qui peuvent être associé à des inserts (slots) dans son DOM fantôme.

Implémentation

Les web composants sont largement supportés par l'ensemble des navigateurs, à l'exception des components built-in qui ne le sont pas nativement sur Safari. Cependant, ils peuvent l'être via un polyfill.
