Project.

Besoin

Réaliser un produit (ou une nouvelle version d'un produit)

Analyse

Un projet est caractérisé par :

  • un périmètre fonctionnel :
    • défini par la MOA (expression des besoins)
    • réalisé par la MOE
  • une qualité, généralement assurée au travers d'une méthode (processus de développement)
  • un coût
    • direct
      • main d'œuvre
        • pour la conception (MOA et MOE)
        • pour la réalisation
      • logiciels
        • outils de génie logiciel (conception, implémentation, tests, documentation automatique)
        • environnement de production (serveurs d'applications, middlewares, composants payants, outils de supervision, etc.)
      • déploiement (par les MOA, MOE, équipes système et réseaux et équipes d'exploitation)
      • matériel (serveurs, sécurité, salles blanches, etc.)
      • formation et supports associés livrables
      • maintenance
        • corrective pour les incidents
        • évolutive (ponctuelles ou quasi-projets)
    • indirect, induit par :
      • les dépassements (forfait, régie forfaitée)
      • l'inadéquation au besoin
      • les changements fonctionnels
      • les défauts de qualité
      • le roulement (turnover) des équipes
  • une durée (au sens date début/date fin) et un type de contrat :
    • Forfait
    • Régie (assistance technique)
      • non forfaitée
      • forfaitée

Conception

Les activités du projet peuvent être décomposées en disciplines.

  • Formalisme (UML)
  • Méthode
    • UP
    • Agiles (XP)
    • Merise
  • Qualité (CMM, ISO 9000)
  • Contrat : forfait, régie
